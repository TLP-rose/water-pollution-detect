# 团队工作日志书写与文件管理规范

## 一、文件管理规则

### 1. 总存放位置
所有工作日志统一存放在：

`docs/work_logs/`

### 2. 周次文件夹
按周建立文件夹，命名格式统一为：

`weekX(起始日期-结束日期)`

例如：

`week1(4.13-4.19)`
`week2(4.20-4.26)`

### 3. 成员文件夹
每周文件夹下，按成员分别建立文件夹：

- `HZH`
- `LBY`
- `XYH`

分别对应：

- HZH：何智慧
- LBY：连冰月
- XYH：肖昱虹

### 4. 日志文件命名
每个人在自己的文件夹下，按日期建立 Markdown 文件，命名格式统一为：

`YYYY.M.D.md`

例如：

`2026.4.13.md`
`2026.4.15.md`

### 5. 文件管理要求
1. 每个人只能在自己的日志文件夹内新增和修改日志文件。
2. 日志文件统一使用 `.md` 格式。（如果还不会写markdown文件的，请自行学习）
3. 文件命名尽量统一，不随意改格式。
4. 日志按周整理，避免所有日志堆在同一个文件夹里。
5. 如需补写日志，仍按实际日期命名后放入对应周文件夹。

---

## 二、工作日志模板

每篇工作日志统一使用以下格式：

``` 
# 工作日志 2026.4.13

# 负责人：何智慧

## 时间
2026.4.13

## 本阶段完成内容
1. 
2. 
3.
## 本阶段结果
## 情绪/压力释放区（可选）
```


---

## 三、说明
1. 每篇日志需包含：标题、负责人、时间、本阶段完成内容、本阶段结果。
2. 情绪/压力释放区为可选内容。
3. 如填写情绪/压力释放区，统一放在最后即可，内容不限。
